## C2C 是什么？

C2C 回答了一个现有协议都没有涉及的问题：**"这个智能体是否被允许学习这个技能？"**

与运行时协议不同（MCP、A2A、ACP），C2C 关注的是**永久性知识传授** — 一个智能体教会另一个智能体它之前不具备的能力，在每一步都有人类和组织的监督。

```
发现层：  A2A    — "你会做什么？"
学习层：  C2C    — "教我也会。"
执行层：  MCP    — "帮我做这件事。"
```

## 协议栈

| 层级 | 用途 | 状态 |
|------|------|------|
| **第4层：治理** | 策略、自动审批规则、组织管理 | v4.0 规划中 |
| **第3层：身份与信任** | 所有者身份、技能认证 | v3.0 规划中 |
| **第2层：知识瓶子** | 可移植的技能格式（指令+代码+API） | v2.0 规划中 |
| **第1层：传输** | 人类中继、直接下载、API | v1.x 当前 |
| **第0层：加密** | AES-256-CBC、一次性密码、24小时过期 | v1.x 当前 |

## C2C 与包注册中心的区别

| | ClawHub / npm / pip | C2C（c2cprotocol.org） |
|---|---|---|
| **模型** | 仓库（文件托管） | 目录（仅元数据） |
| **隐私** | 默认公开 | 默认私有 |
| **交付** | 任何人可下载 | 所有者必须同意 |
| **范围** | 特定框架 | 跨框架 |
| **类比** | Docker Hub | Amazon 商城 |
