# Claw-to-Claw (C2C) 协议

**受治理的 AI 智能体互操作协议 — 信任、学习、协作。**

[English](README.md) | [中文](README_CN.md)

---

## 什么是 C2C？

C2C 定义了 AI 智能体如何建立信任、传授技能和进行协作——在每一步都有人类和组织的监督。

```
发现层：  A2A    — "你能做什么？"
学习层：  C2C    — "教我也会这个。"      （子协议 1）
协作层：  C2C    — "我们一起合作吧。"    （子协议 2）
执行层：  MCP    — "帮我做这件事。"
```

## 协议结构

C2C v2.0 组织为主协议 + 子协议架构：

| 组件 | 用途 | 状态 |
|------|------|------|
| **共享基础** | 身份、信任、治理、安全、加密 | v1.x 当前 |
| **子协议 1：技能传授** | 通过知识瓶子在智能体间传授技能 | v1.x 当前 |
| **子协议 2：可信协作** | 发现可信伙伴并发起协作 | v2.0 新增 |

### 子协议 1：技能传授

> "这个智能体是否应该被允许学习这个技能？"

将 AI 技能打包、加密并在智能体间传输。参考实现：[Lobster Distill](https://github.com/c2c-protocol/lobster-distill)。

### 子协议 2：可信协作

> "哪些可信的智能体可以跟我协作这个任务？"

使 bot 能够发现可信伙伴的可用性、自主选择协作对象并发起协作——公共节点仅提供可见性，不做决策。

**核心原则：公共节点是传呼机，不是调度器。**
